왜 — 그릇과 알맹이
바당(LMS)이 비워 둔 세 가지, 그리고 그것을 채울 계층의 정의와 위치.
출발점 — 바당(AIEP) 진단

그릇은 완성됐다 — 세 가지가 비어 있다

①
학생용 생성형 AI가 없다 — 교육청 현안 문서가 명시: "AI모델 부재(2차 구축 시 개발 필요)".교육부(KERIS) 교육특화 LLM은 별도 트랙 — 시점·범위 불확실
②
송수신 데이터를 확보하지 못한다 — SSO로 연결만 할 뿐, 학생이 AI와 주고받은 전문(프롬프트·응답)은 벤더에 남고 교육청에는 없다.
③
안전·거버넌스 장치가 없다 — 교육청이 직접 안전필터·로깅·통제할 수단 부재. 벤더 앱 직접 사용은 통제·기록 자체가 불가능.
바당은 LMS(그릇)로서는 완성 — 진단 전체는 전편 「바당(AIEP) 현황과 개선 방안」 참조. 바당2는 정확히 이 세 공백을 메운다.
바당2란 — 정의와 위치

재구축이 아니라, 그릇 위에 얹는 ‘AI 계층’이다

  • 바당은 그대로 둔다 — 소스코드 복제·분기(fork)가 아니라, 이미 갖춰진 개방형 표준에 밖에서 끼우는(plug-in) 외부 도구.NIA·타 시·도와 코드 충돌·중복 구축 없음
  • 제주 전용 · 직접 소유 — 11개 시·도 공동(다자 합의)과 달리, AI 변화에 맞춰 제주가 기민하게 확장·변경.
  • 수업을 넘어 업무까지 — 학생·교사 수업 + 교직원 행정 업무를 동일 게이트웨이로.단, 교직원은 학생과 권한·안전·보안정책 분리
포크(복제)가 아니라 표준 연계 — 바당 코드는 건드리지 않는다
핵심 가치 — 데이터 선순환

접근 → 활동 → 전량 기록 → 분석 → 맞춤 환류

단순 AI 챗봇이 아니다 — 모든 활동이 코어 게이트웨이를 경유해 기록·분석·환류된다
기록은 제주가 소유송수신 전문 원본은 교육청 DB에 국내 보관 — 벤더에 남지 않는다.

설계 — 모든 길은 관문을 지난다
게이트웨이 독점이 성립해야 기록·안전·통제가 성립한다 — 그 대가도 설계에 넣었다.
설계 원칙 ① — 게이트웨이 독점

모든 AI 사용이 한 길목을 지난다

벤더 앱 직접 사용(A안)으로는 송수신 전문 확보·통제가 불가능 — 독점(B안)이 본 사업의 직접 근거
설계 원칙 ② — 독점의 대가

한 길목에 모이면, 가용성과 품질이 값이다

가용성 — 단일 장애점(SPOF) 방어

  • 코어 HA·무중단 배포·오토스케일
  • 외부 LLM 장애 → 타 벤더 폴백 · 기능 축소(degrade) 범위 사전 정의 · 서킷브레이커
  • 분기 1회 이상 폴백 전환 훈련 — SLA로 보장(RFP 필수)

우회 — 품질이 낮으면 독점이 무너진다

  • 학생이 개인기기·무료 벤더앱으로 우회하면 로깅 독점 붕괴
  • 과제·평가는 바당2 경유만 인정(세션 토큰·산출물 표식) + MDM·네트워크 정책 연계
  • 기술만으론 100% 불가 — 정책 + 기술 병행
“안전·로깅은 못 끄되, 추론은 멈출 수 있다”는 실패 모드를 인정하고 설계한다.
설계 원칙 ③ — 대원칙(불변)

새 모델·기능은 ‘코어 수정 없이’ 붙인다

  • 코어는 얇고 고정 — 인증·라우터·안전필터·전량 로깅·바당 연계, 길목 기능만. 구체 구현(어떤 모델·기능)은 코어에 박지 않는다.
  • 모델 = 어댑터 — 새 벤더·모델(Claude·GPT·국산·교육부 특화 LLM)은 어댑터 추가 + 설정 등록으로.코어 라우터 무수정
  • 기능 = 확장 슬롯 — 교육 도구는 LTI, AI 도구·데이터는 MCP 표준으로. 새 규격 발명 없이 확장.
  • 코어도 늙는다 — 확장 마켓은 코어 기능의 ‘더 나은 대안’까지 허용, 무엇을 쓸지는 학교·교사가 선택.코어가 낡아도 플랫폼은 늙지 않는다
  • 단 하나의 불변 — 어떤 어댑터·모듈·확장이 붙든 반드시 코어 게이트웨이를 경유. 로깅·안전·개인정보는 우회할 수 없다.
설계 원칙 ④ — 개인정보, 신뢰의 조건

벤더에는 가명 토큰만 — 기록은 감시가 아니다

  • 전송 전 비식별 — 규칙(정규식)+경량 NER 온프레미스 엔진. 결정적·재현가능·감사가능, CPU급 동작.매핑 보유로 법적으로는 ‘가명정보’ — 익명이 아니며 안전조치 의무(§28-4) 적용
  • 매핑 테이블 = 최우선 보안자산 — 유일한 재식별 통로. KMS/HSM 격리 · 2인 승인 · 전건 감사로그.
  • 벤더 계약(DPA) 4대 필수조항 — 학습 미사용 · 제로 리텐션 · 국외 이전 제한 · 침해 통지. 미제공 벤더는 어댑터 등록 불가.
  • 기록 ≠ 상시 열람 — 교사 열람은 집계·요약 기본, 원문은 필요최소·사유 기록. 학습 중 질문·오답·시행착오는 평가에 불리하게 쓰지 않는다.학생이 안심하고 물어야 위기감지·초개별화의 입력이 유지된다

기능 2 — 성패의 전제

모델은 API로 평준화된다 — 승부는 인터페이스다

‘API만 붙인’ 서비스의 5대 실패

  • ① 스트리밍 없음 — 빈 화면 → 이탈
  • ② 마크다운·표·수식·코드 렌더링 부실
  • ③ 대화 맥락 관리 실패 · ④ 결과물 활용 불가
  • ⑤ 모바일 미대응 — 교육용 AI가 파일럿만 하고 사라지는 주된 이유

바당2의 추격 체제

  • 검증된 오픈소스 챗 클라이언트 기반 + 고유부(게이트웨이·안전·LTI)는 플러그인으로 격리
  • 월 주기 릴리스 — 벤더 앱 기능 격차 상시 점검·반영
  • 에듀테크소프트랩 교육연구사 + 업체의 상설 개선 체제(구축과 분리된 지속 개선 계약)
품질이 낮으면 학생은 벤더 앱으로 우회하고 전량 로깅이 무너진다 — UI/UX 품질이 사업 성패의 전제.
기능 10 — 확장 마켓과 생태계

공공은 토대, 민간은 속도 — 이용료는 전액 개발자에게

시범 단계는 무료·선언형(LTI·MCP) 확장만 — 코드 실행형·수익모델은 전면시행 단계(샌드박스·보안 심사 전제)
교육청은 절차상 빠른 기능 변경이 어렵다 — 공공은 안정적 토대(게이트웨이·표준·안전)에 충실하고, 빠르게 바뀌는 응용은 민간의 속도에 맡긴다.
